⚠️ The Problem Before This Stage

Before Phase 1, there was absolutely zero infrastructure. AI Agents cannot reason about healthcare workflows if there is no database schema mapping Patients, Claims, and Providers. An AI needs structured data to analyze.

🛠️ What We Built

💾 PostgreSQL Database

Designed the foundational healthcare data schema using SQLAlchemy (Patients, Claims, Providers).

⚡ FastAPI Backend

Created REST APIs to seed the database, serve dashboard metrics, and expose entity data.

🖥️ React Operations Dashboard

Built a modern frontend using Vite and React to visualize our healthcare entities and provide a workspace for analysts.

🐳 Docker Orchestration

Containerized the entire stack via `docker-compose.yml` so it spins up identically on any machine.

🎙️ Interview Q&A

Q: Why build a database first before adding AI agents?
A: Agentic AI systems are only as smart as the data they can reason over. Without a structured database schema representing the business reality, agents have no ground truth to base their decisions on.
Q: Why did you choose FastAPI for the backend?
A: FastAPI uses Pydantic for data validation, which maps perfectly to strict JSON schemas—something that is heavily required when building structured outputs for AI agents later on.
