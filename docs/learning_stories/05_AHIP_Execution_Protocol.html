1. Execution Philosophy

AHIP follows the Real Rails model: learn the domain, understand the workflow, use AI as an engineering assistant, validate everything, document decisions and deliver phase-wise progress.

2. Intern Learning Model

Learning TrackWhat Intern Learns
Healthcare DomainClaims, provider contracts, benefits, payer workflows, compliance
Agentic AIAgents, memory, context, orchestration, decision outputs
EngineeringAPIs, database, frontend, backend, testing, deployment
Product ThinkingUser personas, SaaS value, dashboards, demo story
GovernanceAuditability, role-based access, explainability, handover

3. Mandatory AI Protocol

  1. Start with the current phase document.
  2. Feed the Product Bible, Architecture Guide and Execution Protocol.
  3. Generate or update Repomix from the current repository.
  4. Ask AI to analyze current state before asking it to code.
  5. Ask for a plan.
  6. Apply changes in small steps.
  7. Run and validate.
  8. Document completion.

4. Required Prompt Pattern

You are helping me build AHIP – AI Healthcare Intelligence Platform.
This is Phase X: [Phase Name].
Use the attached AHIP Product Bible, Architecture Guide, Execution Protocol and latest Repomix.
First analyze the current code and phase goal.
Then provide:
1. What already exists
2. What needs to be added
3. Files to modify
4. Step-by-step implementation plan
5. Validation checklist
Do not convert this into a chatbot or generic RAG system.

5. Phase Completion Standard

  • Phase deliverables implemented or researched.
  • Application runs without blocking errors.
  • README updated.
  • GitHub discussion completion post added.
  • Screenshots/logs attached where needed.
  • Known limitations documented.

6. Troubleshooting Standard

Before reporting a problem, the intern must provide:

  • Current phase
  • Task attempted
  • Error message
  • Screenshot or log
  • What was already tried
  • Latest Repomix or relevant file context
  • AI debugging response

7. Handover Standard

Each phase must leave the project easier for the next person to understand. No hidden work, no undocumented assumptions and no unexplained AI-generated code.
