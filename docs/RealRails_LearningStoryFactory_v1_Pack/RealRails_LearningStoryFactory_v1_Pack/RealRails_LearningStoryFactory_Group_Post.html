Real Rails – Stage Learning Story Factory

Purpose: After every stage or phase, interns must generate one animated HTML learning story to explain what they built, what they learned and how the system works.

Why we are adding this

Some interns are able to build the solution correctly, but struggle to explain what they learned. From now on, every completed stage should produce not only code and GitHub notes, but also a short interactive HTML story that helps the intern explain the stage clearly.

What to generate after each stage

OutputPurpose
Code / ImplementationShows the engineering work is completed.
GitHub Completion PostRecords progress, validation and limitations.
Animated HTML Learning StoryHelps the intern understand and explain the phase like a product demo.

Required Inputs

  • Product Bible
  • Latest Repomix
  • Stage / Phase completion notes
  • Screenshots if available
  • API responses if available
  • Known limitations
  • Next stage details

Prompt to use

# RealRails Learning Story Factory – Intern Execution Prompt

Use this prompt after completing every stage or phase.

## Purpose

Generate an animated, interactive HTML learning story that explains what I completed in this stage, what I learned, how the implementation works and how I can demo it to someone else.

This is not normal documentation.
This should feel like a short interactive learning movie.

## What I Need

Generate one self-contained animated HTML file that helps me explain this stage to:

- Mentor
- Interviewer
- College faculty
- Teammates
- Future interns
- Client/demo audience

The HTML must include:

1. Cinematic opening
2. What problem existed before this stage
3. What we built in this stage
4. Before vs after comparison
5. Animated architecture/workflow diagram
6. Real implementation highlights from Repomix
7. Agentic AI concepts learned
8. Domain concepts learned
9. Demo script
10. Interview questions and answers
11. Quick recap
12. Next stage trailer

---

## Output Rules

- Create a single self-contained HTML file.
- Use inline CSS and JavaScript.
- No external dependencies.
- Use dark modern Real Rails style.
- Make it visual, interactive and easy to present.
- Do not invent implementation details.
- Do not make it boring.
- Do not make it a long report.
- Keep it suitable for a 5–8 minute explanation.

---

## Important

If the current phase is about agents, context, memory, collaboration, decision intelligence, LangGraph or LLM, explain the concept in simple language using the actual implementation.

The goal is:

"Even if I did not fully understand the phase while building it, this HTML should help me understand and explain it confidently."

Expected HTML Sections

  • Cinematic opening
  • Problem before this stage
  • What was built
  • Before vs After
  • Animated workflow / architecture diagram
  • Implementation highlights
  • Concepts learned
  • Demo script
  • Interview Q&A
  • Quick recap
  • Next stage trailer

Quality Standard

The HTML should not look like a boring report. It should help the intern confidently explain the project to a mentor, interviewer, college faculty, teammate or client.

Final Rule

If the intern cannot explain the phase clearly, the phase is not fully complete. The Learning Story HTML is part of the learning evidence.
