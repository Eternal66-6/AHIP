Purpose

AHIP is event-driven. Workflow events trigger agents. This catalog defines sample events for the MVP.

EventDescriptionTriggered AgentsPossible Output
CLAIM_SUBMITTEDNew claim enters workflowClaims Review Agent, Benefit Validation AgentInitial claim risk review
CLAIM_PENDEDClaim requires manual or system reviewClaims Review Agent, Compliance AgentMissing data / documentation alert
PROVIDER_CONTRACT_MISSINGNo valid contract mapping foundProvider Contract Agent, Escalation AgentContract validation task
BENEFIT_AUTH_REQUIREDBenefit rule indicates authorization requirementBenefit Validation Agent, Compliance AgentAuthorization missing alert
CARE_TASK_OVERDUEFollow-up or care task due date missedCare Gap Agent, Patient Journey AgentCare gap alert
DOCUMENT_MISSINGRequired document missingCompliance AgentCompliance gap
CASE_AGED_OVER_THRESHOLDCase open beyond allowed thresholdEscalation AgentEscalation recommendation
USER_OVERRIDE_RECOMMENDATIONUser overrides agent recommendationCompliance AgentOverride audit record

Event Design Rule

Each event should include timestamp, case ID, entity type, event type, source, payload summary and created by.
