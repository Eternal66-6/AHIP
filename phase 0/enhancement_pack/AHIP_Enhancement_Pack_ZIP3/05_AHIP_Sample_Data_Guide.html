Purpose

The AHIP MVP should use realistic but fictional sample data. Do not use real patient data.

Demo Scenarios To Include

  • Claim pended because provider contract is missing.
  • Claim requires prior authorization but document is missing.
  • Patient follow-up task overdue.
  • Case aged beyond threshold and requires escalation.
  • Agent recommendation overridden by user with reason.

Important: Use fictional names and fictional IDs. Do not use real PHI or patient information.
